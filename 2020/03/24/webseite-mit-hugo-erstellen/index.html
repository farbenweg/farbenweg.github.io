Webseite mit Hugo erstellen

24.03.2020

Das ist mein erster Post. Heute ist ein sonniger Märztag mit einer kalten Bise. Wie erstellen wir eine Webseite mit Hugo und Github?

Wir benutzen die drei Komponenten Hugo, Windows PowerShell ISE, Visual Studio Code und Github. Chris hat einen bebilderten Artikel zum Einstieg in Github-Seiten verfasst.

Eine Hugo-Seite erstellen

In der Windows PowerShell erstellen wir den Ordner für die neue Hugo-Seite. In meinem Fall heißt der Ordner “blog-lithium”:

md blog-lithium
cd blog-lithium
hugo new site ./

Automatisch sind die Ordner archetypes, content, data, layouts, resources, static und themes und die Datei config.toml entstanden. Sie sind typisch für eine Hugo-Webseite. Als zweites passen wir die config.toml an. Die baseURL entspricht dem lokalen Server oder deiner Domain.

baseURL = "http://localhost:1313/"
languageCode = "de-de"
title = "Farbenweg"

Eine leere Seite ist generiert.

Ein Hugo-Theme herunterladen

Um ein Theme zu installieren, wechseln wir ins entsprechende Verzeichnis.

cd themes
git clone https://github.com/jrutheiser/hugo-lithium-theme
cd ../

Das Theme kann in lithium umbenannt und in der Konfigurationsdatei config.toml als solches festgelegt werden:

baseURL = "http://localhost:1313/"
languageCode = "de-de"
title = "Beispiel Hugo"
theme = "lithium"

Dann kopieren wir die Inhalte von ExampleSite in unser Seiten-Verzeichnis.

Einen Hugo-Inhalt erstellen

Um einen ersten Blogeintrag zu erstellen, verwenden wir den Befehl hugo new post/___.md:

hugo new post/Webseite-mit-Hugo-erstellen.md

Es entsteht im Unterverzeichnis content/post/ eine Seite, in die wir einen Eintrag schreiben können:

---
title: "Webseite mit Hugo erstellen"
date: 2020-03-24T09:51:12+01:00
draft: true
---
Das ist mein erster Post. Heute ist ein sonniger Märztag mit einer kalten Bise.

Den Post können wir mit Markdown schreiben. Das ist eine Auszeichungssprache mit guter Lesbarkeit im Vergleich zu HTML. Außerdem fügen wir jetzt ein Bild ein. Wir legen es in das Verzeichnis static/images.

![Rinde](/images/rinde2.jpg)

Um die Website anzuschauen starten wir

hugo -D serve

und gehen auf http://localhost:1313/. Die Webseite und das Bild werden angezeigt.

Webseite auf Domain ziehen

Um die Website auf einer Github-Domain anzuzeigen, gehen wir folgendermaßen vor.

Bei der ersten Verwendung des Github-Repositorys klonen wir das Repo in den Public-Ordner:

git clone https://github.com/farbenweg/farbenweg.github.io public

Bei den weiteren Änderungen ist das Klonen nicht mehr nötig. Dann genügen die fünf folgenden Zeilen, um Änderungen auf die Domain zu ziehen:

hugo
cd public
git add --all
git commit -m "inital commit"
git push origin master

Um den Post zu sehen, stellen wir im Bereich oberhalb des Post (Front Matter):

draft: false

Und in der config.toml wählen wir die neue baseURL:

baseURL = "https://farbenweg.github.io/"

Das Über funktioniert

Schon seit einiger Zeit fragte ich mich, ob in Hugo deutsche Page-Titel funktionieren. Vor allem für die Über-Seite war es mir wichtig. Die Lösung habe ich heute hier gefunden:

https://discourse.gohugo.io/t/make-url-use-only-ascii-characters/7592

Das heißt, es braucht nur eine Zeile in der config.toml:

RemovePathAccents = true

Im Titel ist jetzt ein Über anstelle von Ueber zu sehen! In der URL ist einfach ein u anstelle des ü:

https://farbenweg.github.io/uber.html

Und entsprechend sieht es für den Titel dieses Posts aus.

Neues Layout

Das neue Layout umfasst

  • neue Schriftarten,
    • Merienda für die Titel,
    • Lato für die Texte,
  • grüne Titel
  • einen höheren und hellgrünen Header,
  • ein größeres Logo im Header,
  • grüne Schrift beim Überfahren der Navigation.

Der Code in /themes/lithium/layouts/partials/head.html für die neue Schriftart lautet:

<link href='https://fonts.googleapis.com/css?family=Merienda' rel='stylesheet'>

In /themes/lithium/static/css/main.css gab es viele kleine Änderungen für das neue Layout.

Neu auf Github laden

Nach längerem Testen haben sich im Github-Ordner eine Menge angesammelt, die nicht mehr gebraucht werden. Deshalb möchte ich diese Daten löschen. Wenn ich das Theme wechsle, gehe ich ebenfalls so vor, um das Repository ganz zu leeren. Ich wechsle in den Ordner: farbenweg\blog-lithium\public [master ≡]>

git pull
git rm -r .
git commit -m "leer"
git push

Jetzt ist das Repository leer und ich kann die aktuelle Seite hochladen.

Auf andere Domain laden

Um eine andere Domain zu verwenden, klonen wir das entsprechende Repository in unseren public-Ordner.

git clone https://github.com/farbenweg/farbenweg.ch public

Eventuell löschen wir die alte Seite zuerst:

cd public
git rm -r .
git commit -m "leer"
git push

Dann kompilieren wir die Seite und ziehen sie auf die Domain:

hugo
cd public
git add --all
git commit -m "inital commit"
git push origin master

Jetzt sind alle Dateien auf der Domain. Leider ist nichts im Browser zu sehen. Wir brauchen die wichtige Datei CNAME. Sie enthält den Domainnamen, wenn wir eine beliebige Domain wählen. In meinem Fall steht in der Datei CNAME einfach:

farbenweg.ch

Nun funktioniert die Hugo-Seite auf der Domain farbenweg.ch.

Favicon

Jedes Gerät und Betriebssystem verlangt sein eigenes Favicon. Deshalb erstellen wir das Favicon am besten in verschiedenen Grössen und Dateiformaten. Die ganz kleinen Favicons nennen sich favicon.ico oder favicon.png und befinden sich oben im Fenster oder im Tab. Sie sind 16 x 16 Pixel, 32 x 32 Pixel oder 48 x 48 Pixel klein:

favicon.ico

Dann gibt es etwas größere Icons, die sich touch-icon.png oder apple-icon.png nennen. Sie werden in den mobilen Browsern verwendet oder in den Favoriten oder Lesezeichen im Browser.

Da auf meinem alten Smartphone (Samsung Galaxy S4 mini) im Browser das Favicon nicht funktioniert, wird meine Suche nach einer Lösung immer obsessiver. Ich finde immer mehr Größen und Typen von möglichen Favicons für die diversen Geräte und Betriebssysteme.
