Dran bleiben und ein Helikopter

25.03.2020

Das ist mein zweiter Post. Es ist wichtig, dass ich kontinuierlich neue Einträge mache und ausprobiere. Ich versuche es mit einer Aufzählung. Im Moment nehme ich Folgendes wahr:

  • Ich höre einen Helikopter, der einen Baum abtransportiert, der gefällt wird.
  • Ich sehe ein sauberes Zimmer ohne Ameisen.
  • Ich rieche die frische, kühle Frühlingsluft.
  • Ich schmecke die getrockenete Mango in meinem Mund.
  • Ich spüre meine Kleider.
  • Der Helikopter dominiert mit seiner Lautstärke.

Um die Website auf meiner Github Domain anzuzeigen, gehe ich folgendermassen vor:

rmdir public
git clone https://github.com/farbenweg/farbenweg.github.io public
hugo
cd public
git add --all
git commit -m "inital commit"
git push origin master

Dann versuche ich https://farbenweg.github.io/ und es ist nur ein unformatierter Text zu sehen. Es ist nicht das erste Mal, dass ich enttäuscht abbreche. Doch dann ändere ich zwei Dinge: Die Posts werden auf

draft: false

gestellt. Und die config.toml wird auf eine andere baseURL gelegt:

baseURL = "https://farbenweg.github.io/"

Es klappt nicht gleich. Ein paar Minuten später sehe ich auf meinem Smartphone, dass die Posts jetzt alle da sind! Es braucht einfach etwas Zeit, bis sie auf der Domain ankommen.

Jetzt ändere ich das Icon oben links bei der Webseite. Es befindet sich im Ordner themes\lithium\static\images\logo.png. Ich ersetze das Lithium-Logo durch ein Mandala.

In config.toml ändere ich die Menüeinträge:

[[menu.main]]
    name = "Farbenweg"
    url = "/farbenweg/"
[[menu.main]]
    name = "Atelier"
    url = "/atelier/"
[[menu.main]]
    name = "Über"
    url = "/ueber/"

In content/page/ erstelle ich die drei Einträge für die Seiten: Farbenweg.md, Atelier.md und Ueber.md. Diese Seiten können mit Inhalt gefüllt werden.
